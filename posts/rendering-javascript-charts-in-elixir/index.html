Rendering Javascript Charts in Elixir

Mon Jun 8, 2020

If you’re like me, you like to stay away from Javascript when writing your Elixir code. That is why I was pleased to discover chartkick-ex recently when I needed a chart solution for my Phoenix site. It makes embedding Javascript charts in to your Elixir templates a breeze.

To install it, just add the following to your mix.exs file

[
...
{:chartkick, "~>0.4.0"},
{:poison, "~> 3.1"}
]

Chartkick can use Google Charts, Chart.js, or Highcharts. So you need to add one of the following to the <head> of your site

<script src="//www.google.com/jsapi"></script>

//or

<script src="https://cdn.jsdelivr.net/npm/chart.js@2.9.3/dist/Chart.min.js"></script>

//or

<script src="https://cdnjs.cloudflare.com/ajax/libs/highcharts/8.1.0/highcharts.min.js"></script>

Then, you can use Elixir in your template files to handle the rest.

<% data = Poison.encode!([[175, 60], [190, 80], [180, 75]]) %>
<%= Chartkick.line_chart data %>
